The configuration system

Configuration system

The Configuration system is used to change the ClassyTcl configuration settings. Keys, mousebuttons, fonts, colors, menu and toolbar definitions and some general settings can be changed.

Configuration can be changed at different levels, making it possible to eg. to change the configuration for either all ClassyTcl applications or for one specific application, for all users (global) or just one specific user (personal). At startup, a ClassyTcl application will parse different levels in the following order:

  1. ClassyTcl default (def): Global configuration settings
    This level contains the most general configuration settings. They apply to any ClassyTcl application, and should never be deleted. However, they can be overruled by any of the other levels. (The configuration files for this level are stored in the conf/init subdirectory of the ClassyTcl installation directory. Icons are stored in the corresponding conf/icons subdirectory.)
  2. ClassyTcl user (user): Personal configuration
    Settings at this level overrule the global, default settings for a particular user. The configuration id stored in the directory .classy/init (or equivalent depending on OS) in the home directory of the user. Icons are stored in the corresponding .classy/icons subdirectory.
  3. Application Default (appdef): Global application specific configuration
    Settings at this level overrule the global, default settings for a particular application. The configuration is stored in files in the conf/init subdirectory of the application directory. Icons are stored in the corresponding conf/icons subdirectory.
  4. Application user (appuser): Personal application specific configuration
    These configuration settings are specific to the current user (Unix) and application. Settings at this level will overrule any other settings for that appliction and user. The settings are stored in the subdirectory .classy-apps/<application name>/init of the users home directory (or equivalent depending on OS). Icons are stored in the corresponding .classy-apps/<application name>/icons subdirectory.

Icons

Icons cannot be changed from the Configuration dialog yet, but they can be changed by replacing the files in the icon configuration directories in the different levels. Icons are searched by name, starting at the appuser level, followed by the appdef, user and def levels. (Icons are retrieved using the command "Classy::geticon iconname".)

Named configurations

Named configurations can be used to store several possible configurations (eg. blue or gray color style, Unix or Windows keys), wich can be selected easily afterwards by using the "Select configuration" button. Named configurations can also be saved at different levels.

The configuration dialogs

The main part of a configuration dialog is the individual configuration window. The contents of this window depends on what is being configured. However, all configuration windows contain an optionmenu showing the location of the current configuration item and a button bar with the following buttons:
  • Save: save the current settings
  • SaveAs: save the current settings on a different level, or as a named configuration
  • Use: use the currently being edited settings
  • Select: select the settings from another level or named configuration
  • Clear: removes the current configuration entry at the current level. This will cause the settings of one of the lower levels to be used.
  • Reconfigure: go through the normal configuration initialisation.
  • Restore: reset currently being edited settings to those in the file
  • Help: show this help text
  • Close: close the dialog

The general configuration dialog can usually be called from the File menu, by invoking an entry labeled "Configure". (The Tcl command to popup the dialog is "Classy::Config dialog".) It shows a tree with the different configuration levels, and the things that can be configured on that level, next the individual configuration window in which changes to the selected configuration block can be made.
